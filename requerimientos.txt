Requerimientos de Usuario: Parqueo
El usuario debe ver un total de Bs 10.00 al calcular exactamente una hora diurna.
El usuario debe ver un total de Bs 6.00 al calcular exactamente una hora nocturna.
El sistema debe mostrar el monto siempre con dos decimales y prefijo “Bs”.
El usuario debe ver un total correcto cuando combina horas diurnas y nocturnas en un mismo día.
El usuario debe ver un total correcto cuando la estadía cruza medianoche (dos días).
El sistema debe aplicar tope máximo de Bs 50 en un mismo día cuando el cálculo exceda ese monto.
El usuario debe ver un desglose que muestre subtotal, tope aplicado y monto final por día.
El usuario debe obtener siempre Bs 80 cuando selecciona “ticket perdido”.
El usuario debe poder ver el total si se quedo mas de 1 dia.
El sistema debe mostrar un mensaje de error si la hora de salida es anterior a la de entrada.
El sistema debe mostrar un mensaje de error claro bajo el formulario cuando los datos son inválidos.
